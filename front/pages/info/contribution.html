Contribution Guide

프로젝트에 기여하는 방법과 가이드라인을 안내합니다.

01 — Overview

기여 개요

War Archive는 오픈 프로젝트입니다. 역사에 관심 있는 누구나 기여할 수 있으며,
아래 가이드라인을 따라 주시면 원활한 협업이 가능합니다.

01

데이터 기여

새로운 전쟁, 인물, 전투, 사료 등의 JSON 데이터를 추가하거나 기존 데이터의 오류를 수정합니다.

02

디자인 기여

UI/UX 개선, 새로운 페이지 디자인, 반응형 레이아웃 최적화 등 스타일 관련 기여를 합니다.

03

기능 기여

검색, 필터링, 인터랙션 등 JavaScript 기능 개선 및 새로운 기능을 추가합니다.

04

문서 기여

README, 가이드 문서, 코드 주석 등 프로젝트 문서화에 기여합니다.

04 — Steps

기여 절차

아래의 단계를 따라 프로젝트에 기여해 주세요.

1

저장소 Fork

GitHub에서 War Archive 저장소를 본인 계정으로 Fork합니다.

2

브랜치 생성

기여 내용에 맞는 브랜치를 생성합니다.
feature/add-battle-of-midway 또는 fix/typo-in-napoleon 형식을 권장합니다.

3

변경 사항 작성

가이드라인에 맞춰 데이터를 추가하거나 코드를 수정합니다.
변경 전 로컬에서 테스트를 진행해 주세요.

4

커밋 & 푸시

명확한 커밋 메시지를 작성합니다.
feat: 미드웨이 해전 데이터 추가
fix: 나폴레옹 생년 오타 수정

5

Pull Request 생성

원본 저장소에 Pull Request를 생성합니다.
변경 내용에 대한 설명을 상세히 작성해 주세요.

6

코드 리뷰 & 머지

리뷰어의 피드백을 반영한 후, 승인되면 메인 브랜치에 머지됩니다.

05 — Conventions

코딩 컨벤션

JSON 데이터

  • 파일명은 소문자 영문으로 작성하며, 단어 사이는 공백으로 구분합니다.
    (예: battle of midway.json)
  • 모든 키 이름은 camelCase를 사용합니다.
    (예: titleKr, keyFigure, combatRecord)
  • 새 데이터를 추가하면 search.json에 자동 등록됩니다.
    (node assets/js/common/generate-index.js 실행 또는 --watch 모드 사용)
  • 출처가 명확한 데이터만 추가하며, references 필드에 반드시 출처를 기재합니다.
  • 문자열 값은 한국어로 작성하는 것을 원칙으로 합니다.
    (id, nameEn 등 식별자는 영문)
  • 배열 필드(sections, specs, variants 등)는 최소 1개 이상의 항목을 포함해야 합니다.
  • 이미지 URL은 Wikimedia Commons 등 저작권이 자유로운 출처를 사용합니다.
  • 빈 값은 빈 문자열 ""로 두고, 키 자체를 삭제하지 않습니다.
  • JSON 파일은 UTF-8 인코딩으로 저장하며, 들여쓰기는 스페이스 2칸을 사용합니다.
  • 위 다운로드 버튼에서 각 카테고리별 양식을 받아 구조를 확인한 후 작성해 주세요.
06 — Rules

주의사항

해주세요

  • 기존 코드 스타일과 일관성을 유지해 주세요.
  • 데이터의 정확성을 확인하고 출처를 명시해 주세요.
  • 커밋 메시지는 변경 내용을 명확히 설명해 주세요.
  • 큰 변경은 Issue를 먼저 생성하여 논의해 주세요.

하지 마세요

  • 검증되지 않은 정보를 추가하지 마세요.
  • 기존 데이터를 이유 없이 삭제하지 마세요.
  • 외부 프레임워크나 라이브러리를 추가하지 마세요.
  • 특정 국가나 집단에 편향된 서술을 하지 마세요.
